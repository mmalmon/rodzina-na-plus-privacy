2. Jakie dane przetwarza aplikacja

Aplikacja może przetwarzać następujące dane:

  • login rodzica lub dziecka,
  • nazwa wyświetlana użytkownika,
  • identyfikatory kont i rodziny,
  • zadania, wykonania zadań, cele, nagrody, kary i głosy,
  • techniczne identyfikatory powiadomień push, w tym tokeny FCM,
  • lokalne ustawienia aplikacji i sesji.

Jeśli użytkownik włączy opcję Zapamiętaj logowanie, aplikacja może przechowywać login i hasło lokalnie w zaszyfrowanych preferencjach Androida (EncryptedSharedPreferences).

3. Gdzie przechowywane są dane

Aplikacja korzysta z modelu hybrydowego:

Lokalnie na urządzeniu:

  • baza Room,
  • preferencje sesji,
  • preferencje aplikacji,
  • zaszyfrowane dane logowania, jeśli użytkownik wybierze ich zapamiętanie.

W usługach Firebase:

  • Firebase Authentication — logowanie kont online,
  • Cloud Firestore — rodzina, profile użytkowników i dane robocze synchronizowane między urządzeniami,
  • Firebase Cloud Messaging — tokeny urządzeń i powiadomienia push.

4. Cel przetwarzania danych

  • umożliwienie logowania rodzica i dzieci,
  • powiązanie użytkowników w ramach jednej rodziny,
  • synchronizacja zadań, postępów, nagród i głosów,
  • wysyłanie powiadomień o zmianach statusu zadań i nagród,
  • zapisanie ustawień aplikacji na urządzeniu użytkownika.

5. Udostępnianie danych

Aplikacja nie sprzedaje danych użytkowników.

Dane są przetwarzane przez zewnętrznych dostawców infrastruktury tylko w zakresie niezbędnym do działania aplikacji:

  • Google Firebase Authentication,
  • Google Cloud Firestore,
  • Firebase Cloud Messaging.

6. Usuwanie danych

Rodzic może usuwać dzieci z rodziny, a właściciel rodziny może usuwać dodatkowego rodzica.

  • usunięcie dziecka usuwa jego profil i dane aplikacyjne z rodziny,
  • usunięcie dodatkowego rodzica usuwa jego profil z rodziny,
  • lokalny cache na urządzeniu może zostać wyczyszczony przez wylogowanie lub odinstalowanie aplikacji.

Usunięcie danych z warstwy aplikacyjnej nie musi oznaczać automatycznego usunięcia technicznego konta z Firebase Authentication, jeśli nie zostało to dodatkowo wdrożone po stronie administracyjnej lub backendowej.

7. Backup i bezpieczeństwo

Aplikacja ma wyłączony automatyczny backup danych użytkownika na Androidzie. Wrażliwe dane sesji i lokalna baza nie są przeznaczone do kopiowania przez mechanizmy systemowego backupu.

Hasła w lokalnej bazie nie są przechowywane wprost — zapisywany jest skrót hasła. Dane opcji „Zapamiętaj logowanie” są przechowywane lokalnie w zaszyfrowanych preferencjach Androida.

8. Dzieci i opiekunowie

Aplikacja jest przeznaczona do użytku rodzinnego pod nadzorem rodzica lub opiekuna. Konta dzieci tworzone są przez rodzica w ramach rodziny.
